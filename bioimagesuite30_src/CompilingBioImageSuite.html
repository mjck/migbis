1. Overview

As stated in the 3.0beta installation page, installing BioImage Suite consists of twp parts.
  1. Installation of a pre-compiled binary bioimagesuite3_base distribution. This includes:
    • The CLapack (v3) numerical library (original source=clapack.tgz.gz)
    • CMake Cross-Platform Build System 2.6.1 (cmake-2.6.1.tar.gz)
    • GDCM Dicom Library 2.0.10 (gdcm-2.0.10.zip)
    • The Insight Toolkit v 3.10 (InsightToolkit-3.10.0.zip)
    • VTK 5.2 (vtk_5.2_src_patched.zip)
    • MYSQL client libraries (mysql-5.0.51a.tar.gz)
    • Tcl/Tk 8.4.19 (tcl8.4.19-src.tar.gz/tk8.4.19-src.tar.gz) with the following extensions:
      • Incr Tcl/Incr Tk 3.3 (itcl3.3.tar.gz/itk3.3.tar.gz) and IWidgets (iwidgets4.0.1.tar.gz)
      • Tcllib 1.10 (tcllib-1.10.tar.gz)
      • Sqlite 3_6_2 (sqlite-3_6_2-tea.tar.gz)
      • Tile 0.8.2 (tile-0.8.2.tar.gz)
      • TkCon 2.4 (tkcon-2.4.tar.gz)
      The original source files for these can be found at this location. Use the same username/password as for downloading BioImage Suite itself. All of the Open Source software listed above compiled without changes on all platforms.

  2. Installation of the BioImage Suite software package itself.
BioImage Suite consists of a mixture of C++ libraries and .tcl code.  The development process is described in the lecture notes from a seminar taught by Xenios Papademetris (Introduction to Programming for Medical Image Analysis with VTK)

The BioImage Source code is released under the GNU General Public License (v2).

2. Compiling BioImage Suite

Notes

Note that the binary bioimagesuite3_base distribution that comes with BioImage Suite is needed for compiling this. While the contents of these are open source, it is easier to use the precompiled versions to avoid unnecessary troubles.

BioImage Suite is an application not a toolkit. While we provide the source code under the terms of the GPL, support for compiling this is limited to this document! A certain familiarity with the UNIX operating system and cmake is assumed. These instructions are for Linux/Mac OS X and on MS-Windows when using the MSYS development environment. If using Visual Studio 2008 see comments in red below. Essentially you need to open a dos box (cmd.exe) and in steps 2-3 below edit the file setpaths.bat and invoke it.

Steps

  1. Place the BioImage Suite source code into a directory we will refer to as ${MYSRC}.
  2. Edit the setpaths.csh or setpaths.sh file to reflect their new location. i.e. in setpaths.csh change the line 
    setenv BASE ${HOME}/src/bioimagesuite

    to something like ${MYSRC}/bioimagesuite_src/bioimagesuite ... use absolute paths.

    On MS-Windows if using Visual Studio edit the file setpaths.bat instead.
  3. Source the appropriate setpaths file (much like when running the software) to set all environment variables i.e. 
    source ${MYSRC}/bioimagesuite/setpaths.csh
    (or setpaths.sh if using one of the sh/ksh/bash shells. or modify setpaths.bat on MS-Windows).

    On MS-Windows, if using Visual Studio, execute the file setpaths.bat instead.
  4. Optionally: On Unix-like setups, set the following environment variables
    • CC - this is the C compiler you want to use (if it is not the default). For example on Centos 5.x we use CC=gcc34.
    • CXX - this is the C++ compiler you want to use (if it is not the default). For example on Centos 5.x we use CC=g++34.
    • CFLAGS - this sets the defaults C options. Often set to either "-fPIC" or "-fPIC -m32" (-m32 forces 32 bit compilation on 64-bit machines).
    • CXXFLAGS - set identical to CFLAGS.
  5. Configure the build
    6. cd ${MYSRC}/build
    ccmake ..
    On MS-Windows type 
    cmakesetup ..
    instead of ccmake.

    You may also need to specify the positions of vtk, itk etc. If you are using the precompiled distributions, this is all taken care of for you. If not you will need to specify

  6. Compile

    7. Simply type make

    On MS-Windows at this point use Visual Studio to build the project BIOIMAGESUITE3.sln.

  7. To run from the source tree build version 
    8. bis.tcl

    To verify that this is the source build click on the "Which" button on the bottom the main bis.tcl application. If it says something like ${MYSRC}/bioimagesuite then all is good.
